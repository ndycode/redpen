SEQUENCE RATIONALE
================================================================================

Content before Mobile:
- Copy defines spacing, wrapping, hierarchy, truncation
- If mobile finalizes before content, content changes break mobile

Failure States after Mobile:
- Error UX is cross-cutting (desktop + mobile)
- Must validate on all layouts before finalizing
- Doing it early = redo work

================================================================================
SCOPE BOUNDARIES
================================================================================

auth-data-safety vs data-consistency:
  - auth: WHO can access
  - data: WHETHER data stays correct

design-tokens vs ui-components vs ui-consistency:
  - tokens: VALUES
  - components: STRUCTURE
  - consistency: CROSS-PAGE patterns

product-copy-voice-guide vs content-density:
  - voice: HOW it sounds
  - density: WHAT it says

ui-performance vs mobile-performance:
  - ui-perf: desktop render
  - mobile-perf: mobile bundle, hydration

ui-accessibility vs mobile-a11y-touch:
  - a11y: keyboard, screen reader
  - touch: tap targets, motor impairment
